## 快速上手

### 1. 打开文件

启动 TRANSVD 后，您会看到文件拖放区域。支持以下方式：

- **拖放** — 直接将视频/音频文件拖入窗口
- **点击** — 点击拖放区域，从系统文件选择器选取
- **打开新文件** — 在操作界面中随时点击右上角 **"Open New"** 按钮

支持的格式：

| 类型 | 格式 |
|------|------|
| 视频 | MP4, WebM, MKV, MOV, AVI, GIF, TS, MTS, M2TS |
| 音频 | MP3, AAC, WAV, OGG, FLAC |

### 2. 选择操作

加载文件后，左侧 **侧边栏** 列出所有可用操作。点击即可切换。

### 3. 处理与保存

- 调整参数后点击主按钮（如 **Trim**、**Convert**）开始处理
- 处理完成后，点击 **"Save"** 或 **"Download"** 按钮保存到本地
- 部分操作（如 Trim、Merge）提供预览链接，可直接在浏览器中查看

> **提示**：TRANSVD 采用**双引擎架构**。桌面版自动启用**原生 ffmpeg**，无需额外加载；浏览器开发模式回退到 ffmpeg.wasm（首次需加载约 31MB 核心）。详见下方 [🔄 FFmpeg 引擎](#-ffmpeg-引擎) 说明。

## 所有操作详解

### ✂️ Trim（裁剪）

截取视频中的一段。

| 参数 | 说明 |
|------|------|
| **Start (s)** | 开始时间（秒） |
| **End (s)** | 结束时间（秒） |

- 拖动滑块或直接输入数值
- 默认以 `-c copy` 方式快速裁剪（不重新编码）
- 适用于去除片头片尾、截取精彩片段

---

### 🎞️ GIF（生成 GIF）

将视频片段转换为动画 GIF。

| 参数 | 说明 |
|------|------|
| **Start Time** | 起始时间（秒） |
| **Duration** | 持续时长（秒，最大 30s） |
| **FPS** | 帧率（5–30，越高越流畅但文件更大） |
| **Width** | 宽度像素（120–720px，高度等比缩放） |

- 采用**两遍编码**（palettegen + paletteuse）以获得最佳色彩质量
- 处理耗时较长，GIF 文件较大时请耐心等待

---

### 🔄 Convert（格式转换）

在不同视频/音频/图片格式之间转换。

| 参数 | 说明 |
|------|------|
| **类别** | Video / Audio / Image 三种目标类型 |
| **目标格式** | 根据类别选择具体格式 |
| **编码方式** | Stream Copy（快速复制）或 Re-encode（重新编码） |

支持的目标格式：

| 类别 | 格式 |
|------|------|
| 视频 | MP4 (H.264), WebM (VP9), MKV, MOV, AVI |
| 音频 | MP3, AAC, WAV, OGG, FLAC |
| 图片 | JPEG, PNG |

- **Stream Copy**：仅封装转换，速度快、质量无损，但不兼容所有格式转换
- **Re-encode**：完整重新编码，兼容性好，速度较慢

---

### 📦 Compress（压缩）

通过调整编码参数减小文件体积。面板提供 CRF（质量系数）滑块：

| 参数 | 说明 |
|------|------|
| **CRF** | 质量系数（0–51），越低质量越好、文件越大。常用范围：17–28 |

- CRF 23 = 默认平衡
- CRF 28 = 较高压缩、文件更小
- 适用于减小文件体积以便传输或存档

---

### 📐 Resize（调整分辨率）

改变视频画面尺寸。

| 参数 | 说明 |
|------|------|
| **宽/高** | 目标分辨率（像素） |

- 保持宽高比或自定义尺寸
- 用于适配不同设备的播放需求

---

### 🎵 Extract Audio（提取音频）

从视频文件中提取音轨。

| 参数 | 说明 |
|------|------|
| **输出格式** | MP3, AAC, WAV, FLAC, Opus |

- 自动剥离视频流，仅保留音频
- 适用于提取背景音乐、播客素材等

---

### 🔇 Mute（静音）

移除视频中的所有音频轨道。

- 一键操作，无参数调节
- 适用于制作无声视频或静音片段

---

### ⚡ Speed（变速）

调整视频播放速度。

| 参数 | 说明 |
|------|------|
| **速度倍率** | 0.25× – 4× |

- 0.5× = 慢放一半速度
- 2× = 快进双倍速度
- 变速时音频会随之调整（使用 atempo 滤波器）

---

### 🔄 Rotate（旋转/翻转）

旋转或翻转视频方向。

| 参数 | 说明 |
|------|------|
| **旋转角度** | 0°, 90°, 180°, 270° |
| **翻转方向** | 水平翻转、垂直翻转 |

- 适用于手机拍摄的竖屏视频纠正方向
- 可用于创意翻转效果

---

### ✂️ Crop（裁剪画面）

裁剪视频的画面区域，移除不需要的边缘。

| 参数 | 说明 |
|------|------|
| **宽/高** | 裁剪区域尺寸 |
| **X/Y 偏移** | 裁剪起点坐标 |

- 适用于去除黑边、水印或无关内容

---

### 🖼️ Thumbnail（生成缩略图）

从视频中截取一帧作为图片。

| 参数 | 说明 |
|------|------|
| **时间点 (s)** | 截图时间（秒） |
| **输出格式** | JPEG 或 PNG |

- 适用于制作视频封面或预览图

---

### ⏪ Reverse（倒放）

将视频反向播放。

- 完整重新编码，处理时间与视频时长成正比
- 适用于创意特效

---

### 🌅 Fade（淡入淡出）

在视频开头或结尾添加淡入/淡出效果。

| 参数 | 说明 |
|------|------|
| **类型** | Fade In（淡入）/ Fade Out（淡出） |
| **时长 (s)** | 过渡效果持续秒数 |
| **应用于** | Video（视频画面）/ Audio（音频）/ Both（两者） |

---

### 🎨 Adjust（画面调节）

调整视频画面的色彩参数。

| 参数 | 范围 | 说明 |
|------|------|------|
| **Brightness** | -1.0 – 1.0 | 亮度 |
| **Contrast** | 0 – 2.0 | 对比度 |
| **Saturation** | 0 – 2.0 | 饱和度 |
| **Gamma** | 0.1 – 2.0 | 伽马校正 |

- 实时预览效果
- 适用于调色和画面校正

---

### 🏷️ Strip Metadata（去除元数据）

移除视频文件中的所有元数据（拍摄信息、GPS 位置、软件标记等）。

- 一键操作
- 适用于隐私保护

---

### 📝 Subtitles（嵌入字幕）

将字幕文件嵌入到视频中。

| 参数 | 说明 |
|------|------|
| **字幕文件** | 选择 SRT、VTT 或 ASS 格式的字幕文件 |

- 以软字幕（mov_text）方式嵌入
- 播放时可由播放器控制开关

---

### 🔊 Volume（音量调整）

调整视频的音量大小。

| 参数 | 说明 |
|------|------|
| **音量倍率** | 0 – 3.0（1.0 = 原始音量） |

- 适用于统一多个视频的音量
- 可放大或减小音频

---

### 🔁 Loop（循环）

将视频片段循环播放多次。

| 参数 | 说明 |
|------|------|
| **循环次数** | 重复播放的次数 |

- 适用于制作循环背景视频或短视频素材

---

### 🖼️ Overlay（叠加水印）

在视频上叠加图片水印/Logo。

| 参数 | 说明 |
|------|------|
| **水印图片** | 选择 PNG、JPG 等图片文件 |
| **位置** | 左上、右上、左下（默认）、右下、居中 |
| **Logo 大小** | 相对原视频的百分比（5%–50%） |

- 适用于为视频添加品牌水印
- 支持抠图透明背景的 PNG

---

### 🎚️ Mix Audio（混音）

将另一个音频文件混入视频中。

| 参数 | 说明 |
|------|------|
| **音频文件** | 选择 MP3、WAV 等音频文件 |
| **混音比例** | 原始音频与外部音频的混合比例 |

- 适用于添加背景音乐
- 可同时保留原声

---

### 🔗 Concat（合并视频）

将两个视频文件首尾拼接为一个。

| 参数 | 说明 |
|------|------|
| **第一个片段** | 当前已加载的视频 |
| **第二个片段** | 点击"Choose File"选择另一个视频 |

- 两个视频最好使用相同的编码格式和分辨率
- 采用 `-c copy` 方式，速度极快

---

### 📺 Side-by-Side（并排对比）

将两个视频并排拼接为同一个画面。

| 参数 | 说明 |
|------|------|
| **第二个视频** | 选择要对比的视频文件 |
| **布局** | 左右并排或上下排列 |

- 适用于视频对比、前后效果展示

---

### 🖥️ PiP（画中画）

将一个视频作为画中画叠加到主视频上。

| 参数 | 说明 |
|------|------|
| **画中画视频** | 选择作为小窗的视频 |
| **位置** | 小窗放置位置（角落） |
| **小窗大小** | 相对主画面的比例 |

- 适用于教程录制、反应视频等

---

### ℹ️ MediaInfo（媒体信息）

查看视频/音频文件的详细编码信息。

| 显示信息 | 说明 |
|---------|------|
| **文件名** | 文件名称 |
| **文件大小** | MB / KB 显示 |
| **编码信息** | 点击"Analyze Media"运行 ffmpeg 分析 |

- 显示视频编码、分辨率、比特率、音频轨道等详细数据
- 使用 ffmpeg 的 `-hide_banner -i input` 命令分析

---

### ⌨️ Raw FFmpeg（原始命令）

**高级功能** — 直接输入自定义 ffmpeg 参数。

预置命令库（一键填入）：

- H.264 Compress (CRF 23)
- H.264 Compress (CRF 28)
- VP9 WebM
- Extract Audio MP3 / WAV
- Scale to 720p / 480p
- GIF (10fps, 320px)
- Remove Metadata
- Grayscale

使用方式：

1. 点击预置按钮快速填充参数，或手动输入
2. 参数语法：`-c:v libx264 -crf 23 -c:a aac`（无需指定 `-i input` 和 `-y output`）
3. 选择输出文件扩展名
4. 点击 **"Run Command"**

> **注意**：此功能面向熟悉 ffmpeg 命令行的用户，错误参数可能导致处理失败。

## 通用功能

### 🔄 FFmpeg 引擎

TRANSVD 采用**双引擎架构**，根据运行环境自动选择最优处理引擎：

```
用户执行操作
    │
    ├─ Tauri 桌面环境 ──→ 原生 ffmpeg sidecar（主路径）
    │                       ├── 直接启动系统 ffmpeg 进程
    │                       ├── 实时解析 stderr 获取处理进度
    │                       ├── 支持 .kill() 取消操作
    │                       └── 无 2GB 内存限制
    │
    └─ 浏览器 / 降级 ──→ ffmpeg.wasm（回退路径）
                        ├── 首次需下载约 31MB WASM 核心
                        └── 受浏览器 ~2GB 内存上限约束
```

| 特性 | 原生 ffmpeg（桌面） | ffmpeg.wasm（降级） |
|------|:---:|:---:|
| 启动速度 | 即时 | 需下载 31MB WASM 核心 |
| 大文件支持 | 无内存限制 | ~2GB 上限 |
| 处理速度 | ⚡ 原生性能 | 中等（WASM 开销） |
| 取消操作 | ✅ 即时 .kill() | ⚠️ 有限支持 |
| 实时进度 | ✅ stderr 逐行解析 | ✅ 支持 |
| 离线可用 | ✅ 是 | ⚠️ 首次需要网络 |

> **桌面用户**：无需关注 WASM 加载——原生 ffmpeg 随应用安装包分发，即开即用。
> **开发者**：在浏览器中运行 `npm run dev` 时会自动切换到 ffmpeg.wasm 模式。

### 📊 处理进度

每个操作执行时会显示进度条和当前进度百分比（0–100%）。

### ❌ 错误处理

如果处理出错，会以红色提示框显示错误信息。常见原因：

- 输入文件损坏或不支持
- 参数设置不当（如结束时间小于开始时间）
- 原生引擎：ffmpeg 进程返回非零退出码（详见错误日志）
- WASM 引擎：内存限制（~2 GB），超大文件请使用桌面原生模式

### 💾 保存文件

- **Tauri 环境**：弹出系统原生"另存为"对话框，选择保存位置
- **浏览器环境**（开发模式）：直接下载到默认下载目录

### 🌐 多语言

应用内置中文和英文界面。点击 Header 上的语言按钮切换。

## 高级功能

### 自定义 FFmpeg 参数

通过 **Raw FFmpeg** 面板，您可以执行任意 ffmpeg 命令组合。推荐从预置库起步，熟悉后再编写自定义参数。

### 文件限制说明

| 限制项 | 原生引擎（桌面） | WASM 引擎（浏览器降级） |
|--------|:---:|:---:|
| **内存限制** | 无（系统内存） | ~2 GB |
| **建议最大文件** | 不限 | < 1 GB |
| **GIF 时长** | 最长 30s | 最长 30s |

- 桌面原生模式下，处理速度取决于 CPU 核心数和磁盘 I/O
- GIF 时长建议控制在 10 秒以内以保持合理处理时间

### 性能建议

- **优先使用桌面原生模式** — 更快、无内存限制、支持取消操作
- GIF 生成使用两遍编码，速度较慢，建议控制时长和分辨率
- 格式转换时优先使用 **Stream Copy**（如果目标格式兼容）
- 合并视频时确保两段视频编码一致，避免重新编码
- 超大文件（>2GB）请在桌面版中操作，避免 WASM 内存瓶颈

## 常见问题

**Q: 桌面版和浏览器开发模式有什么区别？**

| | 桌面版（Tauri 原生） | 浏览器开发模式 |
|------|:---:|:---:|
| 处理引擎 | 原生 ffmpeg（sidecar） | ffmpeg.wasm |
| 文件选择 | 系统原生对话框 | HTML 文件输入 |
| 保存文件 | 系统另存为对话框 | 浏览器下载 |
| 大文件处理 | 无限制 | ~2GB 上限 |
| 启动速度 | 即时 | 需加载 31MB 核心 |

**Q: 应用首次加载很慢？**

**桌面版**：没有加载过程——原生 ffmpeg 已内置在安装包中，即点即用。

**浏览器开发模式**：首次需要下载 ffmpeg.wasm 核心（约 31 MB），取决于网络速度。后续使用无需再次下载。

**Q: 处理大文件时卡住或报错？**

**桌面版**：原生 ffmpeg 无内存限制，可以处理任意大小的文件（受限于可用系统内存和磁盘空间）。

**浏览器开发模式**：ffmpeg.wasm 受 ~2 GB 内存限制。1 GB 以上的文件建议使用桌面版。

**Q: 如何添加更多格式支持？**

TRANSVD 已覆盖主流格式。如需特殊格式（如 HEVC、AV1），请在 Raw FFmpeg 面板中手动指定参数。

**Q: 处理结果不满意怎么办？**

调整参数后重新处理即可。所有操作均为"无损尝试"——原始文件不会被修改。

**Q: 应用支持批量处理吗？**

当前版本为单文件处理模式。可通过 Raw FFmpeg 面板实现简单的批量操作。
